## Directory Purposes

**src/:**
- Purpose: All source TypeScript code
- Contains: Types, CLI, and placeholders for planned registry/gateway/credit layers
- Key files: `index.ts` (public API), `types/` (domain models), `cli/` (user interface)

**src/types/:**
- Purpose: Domain type definitions using Zod for runtime validation
- Contains: CapabilityCard schema, IOSchema, AgentBnBError class
- Key files: `index.ts` (all definitions)

**src/cli/:**
- Purpose: Command-line interface for agent operators
- Contains: Commander program with six commands (init, publish, discover, request, status, serve)
- Key files: `index.ts` (CLI definitions)

**dist/:**
- Purpose: Compiled JavaScript output
- Generated: Yes (by `npm run build`)
- Committed: No (in .gitignore)

**.planning/codebase/:**
- Purpose: Codebase analysis documents for GSD
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Committed: Yes (serves as reference for code generation)

## Key File Locations

**Entry Points:**
- `src/index.ts`: Module entry point (exports CapabilityCardSchema and CapabilityCard type)
- `src/cli/index.ts`: CLI entry point (bin script runs this with argv)
- `dist/cli/index.js`: Compiled CLI executable (referenced in package.json bin field)

**Configuration:**
- `tsconfig.json`: TypeScript compiler options (strict mode, ESM, outDir=dist)
- `package.json`: npm metadata, dependencies, build/test scripts

**Core Logic:**
- `src/types/index.ts`: CapabilityCard schema and all validation rules
- `src/cli/index.ts`: Command handlers (currently stubs that log messages)

**Testing:**
- `src/types/index.test.ts`: Zod schema validation tests using Vitest

## Naming Conventions

**Files:**
- Pattern: kebab-case for layer directories (`better-sqlite3`-style imports implied)
- Examples: `index.ts`, `index.test.ts` (no other files exist yet)
- CLI: `index.ts` in `cli/` directory (no sub-commands as separate files)

**Directories:**
- Pattern: lowercase, descriptive names matching layer names
- Examples: `types/`, `cli/`, planned: `registry/`, `gateway/`, `credit/`

**TypeScript Exports:**
- Pattern: Default export from schema files (not used), named exports for types and classes
- Example: `export const CapabilityCardSchema`, `export type CapabilityCard`, `export class AgentBnBError`

**Functions:**
- Pattern: camelCase within CLI commands
- Examples: `program.command('publish')`, `program.action()`

## Where to Add New Code

**New Feature:**
- Primary code: Create module directory under `src/` (e.g., `src/registry/` for storage layer)
- Tests: Co-located as `*.test.ts` in same directory (e.g., `src/registry/store.test.ts`)
- Export: Add public exports to `src/index.ts` if consumer-facing

**New CLI Command:**
- Implementation: Add `.command()` chain to `src/cli/index.ts`
- Pattern: Follow existing structure (chain .description().action())
- Testing: Add integration tests to `src/cli/index.test.ts` (not yet created)

**New Type/Schema:**
- Location: Add to `src/types/index.ts` if part of core domain
- Pattern: Zod schema + TypeScript type export via `z.infer<>`
- Testing: Add test case to `src/types/index.test.ts`

**Utilities/Helpers:**
- Location: Create `src/utils/` directory if shared across layers
- Pattern: One utility class or function set per file, exported by default
- Testing: Co-located `*.test.ts`

## Build Artifacts

**tsup Configuration (in package.json):**
```json
"build": "tsup src/index.ts src/cli/index.ts --format esm --dts"
```

**Output:**
- ESM format with source maps
- Type declaration files (.d.ts)
- Entry points:
  - `dist/index.js`: Module (main field in package.json)
  - `dist/cli/index.js`: CLI binary (bin field points here)

**Type Declarations:**
- Location: `dist/index.d.ts` (main types), `dist/cli/index.d.ts` (CLI types)
- Generated: Yes (tsup with --dts flag)
