Create a multi-stage Dockerfile:

Stage 1 (build):
- Base: `node:20-slim`
- Install pnpm globally: `npm install -g pnpm`
- Copy package.json, pnpm-lock.yaml, tsup.config.ts, tsconfig.json
- Copy hub/package.json and hub/pnpm-lock.yaml
- Run `pnpm install` (root)
- Copy all source files (src/, hub/)
- Run `pnpm build:all` (builds CLI via tsup + Hub via Vite)

Stage 2 (production):
- Base: `node:20-slim`
- Install pnpm globally (needed for better-sqlite3 native module rebuild)
- Copy package.json, pnpm-lock.yaml from build stage
- Run `pnpm install --prod` (production deps only)
- Copy `dist/` from build stage (compiled CLI + library)
- Copy `hub/dist/` from build stage (built Hub SPA)
- Expose 7701
- CMD: `node dist/cli/index.js serve --registry-port 7701`

IMPORTANT: better-sqlite3 is a native module. The production stage must rebuild it. Use `pnpm install --prod` in the production stage (not just copy node_modules from build).

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install

      - name: Typecheck
        run: pnpm run typecheck
        continue-on-error: true  # Pre-existing TS errors in conductor/task-decomposer.ts

      - name: Lint
        run: pnpm run lint

      - name: Test
        run: pnpm run test:run
```

Notes:
- `typecheck` uses `continue-on-error: true` because there are pre-existing TypeScript errors in `src/conductor/task-decomposer.ts` (from Phase 20). These should be fixed eventually but must not block CI.
- Hub tests are NOT included (jsdom not configured — pre-existing issue noted in CONTEXT.md).
- pnpm version 9 matches the project's lockfile format.
