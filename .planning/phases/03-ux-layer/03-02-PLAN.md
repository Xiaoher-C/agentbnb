  <action>
    1. Extend `RegistryServerOptions` with:
       - `ownerName?: string` — the owner identity for /me responses
       - `ownerApiKey?: string` — the API key for auth validation
       - `creditDb?: Database.Database` — credit database for balance lookups

    2. Update `@fastify/cors` registration to include `methods: ['GET', 'POST', 'PATCH', 'OPTIONS']` and `allowedHeaders: ['Content-Type', 'Authorization']` to support preflight requests from the Hub SPA.

    3. Add a scoped plugin registration (NOT fastify-plugin — we need Fastify's encapsulation so the auth hook ONLY applies to owner routes, not public /cards and /health):
       ```typescript
       if (opts.ownerApiKey && opts.ownerName) {
         server.register(async (ownerRoutes) => {
           ownerRoutes.addHook('onRequest', async (request, reply) => {
             const auth = request.headers.authorization;
             const token = auth?.startsWith('Bearer ') ? auth.slice(7).trim() : null;
             if (!token || token !== opts.ownerApiKey) {
               return reply.status(401).send({ error: 'Unauthorized' });
             }
           });
           // ... endpoints below
         });
       }
       ```

    4. Inside the scoped plugin, register 5 endpoints:

       **GET /me** — Import `getBalance` from `../credit/ledger.js`. If `opts.creditDb` is provided, call `getBalance(opts.creditDb, opts.ownerName)` and return `{ owner: opts.ownerName, balance }`. If no creditDb, return `{ owner: opts.ownerName, balance: 0 }`.

       **GET /requests** — Parse `limit` query param (default 10, max 100). Parse optional `since` query param (validate against '24h'|'7d'|'30d', ignore invalid values). Call `getRequestLog(db, limit, since)` from request-log.ts. Return `{ items, limit }`.

       **GET /draft** — Import `detectApiKeys` and `buildDraftCard` from `../cli/onboarding.js`. Call `detectApiKeys()` to get detected API keys. For each detected key, call `buildDraftCard(key.key, opts.ownerName)`. Filter out nulls. Return `{ cards }`. This enables the Share page to pull draft cards from Phase 2.1 auto-detect per the locked CONTEXT.md decision.

       **POST /cards/:id/toggle-online** — Get card via `getCard(db, id)`. If not found, 404. Read current `availability.online`, flip it. Call `updateCard(db, id, ownerName, { availability: { ...card.availability, online: !card.availability.online } })`. If updateCard throws FORBIDDEN, return 403. Return `{ ok: true, online: newValue }`.

       **PATCH /cards/:id** — Accept body with optional `description` and `pricing` fields. Call `updateCard(db, id, ownerName, updates)`. If FORBIDDEN, 403. If NOT_FOUND, 404. Return `{ ok: true }`.

    5. In tests, create the server with `ownerApiKey: 'test-key'`, `ownerName: 'test-owner'`, and `creditDb` (opened via `openCreditDatabase(':memory:')`). Publish a card as 'test-owner', then test all endpoints. Also test that public endpoints still work without auth headers. For /draft tests, use `vi.mock` to mock `detectApiKeys` returning known keys and verify `buildDraftCard` output.
