    - executes "echo '{\"key\": \"val\"}'" with output_type json and returns parsed {key: "val"}
    - executes command with output_type file and returns trimmed stdout as file path
    - rejects command "rm -rf /" when allowed_commands is ["echo", "python3"]
    - kills process after timeout_ms and returns timeout error
    - uses working_dir as cwd for child process
    - stderr on non-zero exit returns error with stderr content
  </behavior>
  <action>
    1. Create `src/skills/command-executor.ts`:
       - Export class `CommandExecutor` implementing `ExecutorMode`
       - `execute(config: CommandSkillConfig, params)`:
         a. Security check: if config.allowed_commands is set, extract the base command (first word) and verify it's in the allowlist. If not, return `{ success: false, error: "Command not allowed: {cmd}" }`
         b. Interpolate command string: use `interpolate(config.command, { params })` from `src/utils/interpolation.ts`
         c. Execute via `child_process.exec` (async wrapper) with:
            - `timeout`: config.timeout_ms (default 30000)
            - `cwd`: config.working_dir (default process.cwd())
            - `maxBuffer`: 10MB
            - `shell`: true
         d. On non-zero exit: return `{ success: false, error: stderr || "Command failed with exit code N" }`
         e. Parse output based on config.output_type:
            - `text`: return stdout.trim()
            - `json`: JSON.parse(stdout.trim()) — catch parse errors
            - `file`: return { file_path: stdout.trim() }
         f. Return `{ success: true, result: parsedOutput }`

       - Test allowed_commands blocks unauthorized command

       - Test working_dir is passed to exec
       - Test non-zero exit returns stderr
