  <behavior>
    - interpolate("Hello ${params.name}", context) replaces with context.params.name
    - interpolate("${prev.result.text}", context) extracts nested path from context.prev.result.text
    - interpolate("${steps[0].result.audio}", context) extracts from context.steps[0].result.audio
    - Non-string values pass through unchanged
    - Missing paths resolve to empty string (not throw)
    - Multiple ${} expressions in one string all resolve
    - Literal text without ${} returns unchanged
  </behavior>
  <action>
    1. Create `src/utils/interpolation.ts`:
       - Export `interpolate(template: string, context: Record<string, unknown>): string`
         - Regex: `/\$\{([^}]+)\}/g`
         - For each match, resolve the dot-path expression against context
         - Handle array index notation: `steps[0].result` → context.steps[0].result
         - If resolved value is object, JSON.stringify it
         - If path not found, replace with empty string
       - Export `interpolateObject(obj: Record<string, unknown>, context: Record<string, unknown>): Record<string, unknown>`
         - Deep-walk object, interpolate all string values
       - Export `resolvePath(obj: unknown, path: string): unknown`
         - Split on `.` and `[N]`, walk object tree

  <name>Task 2: Pipeline Executor Implementation</name>
  <files>src/skills/pipeline-executor.ts, src/skills/pipeline-executor.test.ts</files>
  <behavior>
    - 2-step pipeline: step 1 result is available as ${prev.result} in step 2
    - 3-step pipeline: step 0 result available as ${steps[0].result} in step 2
    - Command step: executes shell command and captures stdout
    - Step failure: returns error with "Step N failed: {error}" and stops
    - Empty pipeline (no steps) returns success with empty result
  </behavior>
  <action>
    1. Create `src/skills/pipeline-executor.ts`:
       - Export class `PipelineExecutor` implementing `ExecutorMode`
       - Constructor takes a reference to the parent `SkillExecutor` (for dispatching sub-skill calls)
       - `execute(config: PipelineSkillConfig, params)`:
         a. Initialize context: `{ params, steps: [], prev: { result: null } }`
         b. For each step in config.steps:
            - Interpolate step's input_mapping using context
            - If step has `skill_id`: call `this.skillExecutor.execute(skill_id, interpolatedParams)` — if result.success is false, stop and return error
            - If step has `command`: run via `child_process.execSync` or `exec` with timeout, capture stdout
            - Store result in `context.steps[i]` and update `context.prev`
         c. Return `{ success: true, result: context.steps[lastIndex].result }`
       - Use `interpolateObject` from `src/utils/interpolation.ts` for input_mapping resolution

    2. Create `src/skills/pipeline-executor.test.ts`:
       - Mock the SkillExecutor with a fake that returns predefined results per skill_id
       - Test 2-step pipeline with ${prev.result} piping
       - Test ${steps[N].result} reference
       - Test command step (mock child_process.exec or use `echo`)
       - Test step failure stops pipeline
  </action>
