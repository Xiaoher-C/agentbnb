  <name>Task 1: API Executor Implementation</name>
  <files>src/skills/api-executor.ts, src/skills/api-executor.test.ts</files>
  <behavior>
    - POST request sends input_mapping body.* fields as JSON body
    - GET request sends input_mapping query.* fields as URL query params
    - Path params: input_mapping path.* replaces {param} in endpoint URL
    - Header params: input_mapping header.* sent as request headers
    - Auth bearer: sends Authorization: Bearer {token}
    - Auth apikey: sends configured header name with API key value
    - Auth basic: sends Authorization: Basic base64(user:pass)
    - Output mapping: "response.data.audio" extracts nested field from JSON response
    - Retry: on 429/500/503, retries up to config.retries times with exponential backoff (100ms * 2^attempt)
    - Timeout: AbortController kills request after config.timeout_ms
    - Non-2xx response (after retries) returns success:false with status code in error
  </behavior>
  <action>
    1. Create `src/skills/api-executor.ts`:
       - Export class `ApiExecutor` implementing `ExecutorMode`
       - `execute(config: ApiSkillConfig, params)`:
         a. Build URL: replace `{key}` in endpoint with path-mapped params
         b. Build query string from query-mapped params
         c. Build body from body-mapped params
         d. Build headers from header-mapped params + auth headers
         e. Call fetch with AbortController timeout
         f. If response not ok and retryable (429/500/503) and retries remaining, wait and retry
         g. Parse JSON response
         h. Apply output_mapping: dot-notation path extraction (e.g., "response.data.audio" extracts from response body)
         i. Return `{ success: true, result: mappedOutput }`
       - Helper: `applyInputMapping(params, mapping, target)` — routes param to correct location
       - Helper: `extractByPath(obj, dotPath)` — extracts nested value by dot notation
       - Helper: `buildAuthHeaders(auth)` — returns header entries for auth config
