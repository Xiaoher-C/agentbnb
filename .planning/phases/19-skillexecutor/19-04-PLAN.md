    - Webhook channel: POSTs to agent_name's webhook URL with {task, params} and returns parsed response
    - Process channel: spawns `openclaw run {agent_name} --task "{JSON}"` and captures stdout
    - Telegram channel: POSTs to Telegram bot webhook with formatted task message, returns success:true (fire-and-forget for MVP)
    - Timeout: AbortController / process kill after timeout_ms (default 60000)
    - Invalid channel type returns error
    - Non-200 webhook response returns error
  </behavior>
  <action>
    1. Create `src/skills/openclaw-bridge.ts`:
       - Export class `OpenClawBridge` implementing `ExecutorMode`
       - `execute(config: OpenClawSkillConfig, params)`:
         a. Build OpenClaw task payload: `{ task: config.name, params, source: 'agentbnb', skill_id: config.id }`
         b. Switch on config.channel:
            - `webhook`: POST to `http://localhost:3000/openclaw/${config.agent_name}/task` (configurable base URL via config or env). Use fetch with AbortController timeout. Parse JSON response. Return `{ success: true, result: responseBody }`.
            - `process`: Spawn `openclaw run ${config.agent_name} --input '${JSON.stringify(payload)}'` via child_process.execSync with timeout. Parse stdout as JSON. Return result.
            - `telegram`: POST to `https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage` with chat_id from config and formatted task text. Return `{ success: true, result: { sent: true, channel: 'telegram' } }`. (MVP: fire-and-forget, no response capture.)
         c. Map errors to `{ success: false, error: message }`

    2. Create `src/skills/openclaw-bridge.test.ts`:
       - Mock fetch for webhook channel tests
       - Mock child_process for process channel tests
       - Test webhook: correct URL, payload format, response mapping
       - Test process: correct command, stdout parsing
       - Test telegram: fire-and-forget POST
       - Test timeout handling
       - Test invalid channel returns error
