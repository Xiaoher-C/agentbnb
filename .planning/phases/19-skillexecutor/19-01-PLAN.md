  <files>src/skills/skill-config.ts, src/skills/skill-config.test.ts, package.json</files>
  <behavior>
    - parseSkillsFile() with valid 4-mode YAML returns typed SkillConfig[] (api, pipeline, openclaw, command)
    - parseSkillsFile() with missing required fields throws ZodError with path info
    - parseSkillsFile() with unknown type field fails validation
    - Environment variable expansion: "${SOME_VAR}" in string fields expands from process.env
    - parseSkillsFile() handles empty skills array gracefully (returns [])
    - Each skill type has discriminated union by `type` field
  </behavior>
  <action>
    1. Install js-yaml: `pnpm add js-yaml && pnpm add -D @types/js-yaml`

    2. Create `src/skills/skill-config.ts`:
       - Define Zod schemas for each skill type using discriminated union on `type`:
         - `ApiSkillConfigSchema`: id, type:'api', name, endpoint, method (GET/POST/PUT/DELETE), auth (bearer/apikey/basic — optional), input_mapping (Record<string,string>), output_mapping (Record<string,string>), pricing ({credits_per_call}), timeout_ms (optional, default 30000), retries (optional, default 0), provider (optional)
         - `PipelineSkillConfigSchema`: id, type:'pipeline', name, steps[] (each with skill_id or command, input_mapping), pricing
         - `OpenClawSkillConfigSchema`: id, type:'openclaw', name, agent_name, channel (telegram/webhook/process), pricing, timeout_ms (optional)
         - `CommandSkillConfigSchema`: id, type:'command', name, command (string), output_type (json/text/file), allowed_commands (string[] — optional allowlist), working_dir (optional), timeout_ms (optional, default 30000), pricing
       - `SkillConfigSchema` = z.discriminatedUnion('type', [Api, Pipeline, OpenClaw, Command])
       - `SkillsFileSchema` = z.object({ skills: z.array(SkillConfigSchema) })
       - Export `parseSkillsFile(yamlContent: string): SkillConfig[]` — parses YAML, validates with Zod, expands env vars
       - Export `expandEnvVars(value: string): string` — replaces `${VAR_NAME}` with process.env[VAR_NAME], throws if env var undefined and no default
       - Walk all string values in parsed config and expand env vars before Zod validation

    3. Create `src/skills/skill-config.test.ts`:
       - Test all behaviors listed above
       - Use inline YAML strings as test fixtures (no external files)
       - Set process.env in test setup, clean in teardown
