   - If the package does not exist under that name, try `@modelcontextprotocol/sdk` instead. Check npm registry to confirm.

2. Create `src/mcp/server.ts` — the main MCP server module:
   - Export a `McpServerContext` interface holding shared state: `configDir: string`, `config: Config`, `identity: AgentIdentity`, plus lazy-initialized database accessors for registryDb and creditDb (open on first use, close on shutdown).
   - Export `async function startMcpServer()` that:
     a. Loads config via `loadConfig()`. If null, writes an error to stderr and exits.
     b. Ensures identity via `ensureIdentity(configDir, owner)`.
     c. Creates `McpServer({ name: 'agentbnb', version: '<from package.json>' })`.
     d. Creates a `McpServerContext` with config + identity + configDir.
     e. Calls each `registerXxxTool(server, ctx)` function to register all 6 tools.
     f. Creates `StdioServerTransport()` and calls `server.connect(transport)`.
     g. Writes startup info to stderr (NOT stdout — stdout is for MCP JSON-RPC only): `AgentBnB MCP server started (owner: ${identity.owner})`.
   - IMPORTANT: All console.log/console.error MUST go to stderr in MCP servers. stdout is reserved for JSON-RPC protocol messages. Use `process.stderr.write()` or `console.error()` for any logging.

3. Create `src/mcp/tools/discover.ts`:
   - Export `function registerDiscoverTool(server: McpServer, ctx: McpServerContext)`.
   - Register tool `agentbnb_discover` with:
     - description: "Search for agent capabilities on the AgentBnB network. Returns matching capability cards from both local and remote registries."
     - inputSchema: `z.object({ query: z.string().describe('Natural language search query'), level: z.number().optional().describe('Filter by capability level (1=Atomic, 2=Pipeline, 3=Environment)'), online_only: z.boolean().optional().describe('Only show online agents') })`
   - Handler:
     a. Open local registry DB from ctx.config.db_path.
     b. Call `searchCards(db, query)` for local results.
     c. If `ctx.config.registry` is set, call `fetchRemoteCards(registryUrl, { q: query, level, online: online_only })` wrapped in try/catch (remote failure is non-fatal).
     d. Call `mergeResults(localCards, remoteCards, true)` to get deduplicated TaggedCard[].
     e. Close the DB.
     f. Return content as JSON text with the merged results array. For each card, include: id, name/agent_name, owner, description, skills summary, pricing, source (local/remote), availability.online.

4. Create `src/mcp/tools/status.ts`:
   - Export `function registerStatusTool(server: McpServer, ctx: McpServerContext)`.
   - Register tool `agentbnb_status` with:
     - description: "Check your AgentBnB agent status: identity, credit balance, and configuration."
     - inputSchema: `z.object({})` (no inputs needed)
   - Handler:
     a. Get balance: If ctx.config.registry is set, use `createLedger({ registryUrl, ownerPublicKey, privateKey })` then `ledger.getBalance(owner)`. Otherwise use local `getBalance(creditDb, owner)`.
     b. Return JSON with: agent_id, owner, public_key, balance, registry_url (if configured), config_dir.

     d. If ctx.config.registry is set, POST card to `${registryUrl}/cards` (include gateway_url from config).
     e. Return JSON with: success, card_id, card_name, remote_published (boolean).

1. Create `src/mcp/tools/request.ts`:
   - Export `function registerRequestTool(server: McpServer, ctx: McpServerContext)`.
   - Register tool `agentbnb_request` with:
     - description: "Request execution of a skill from another agent on the AgentBnB network. Handles credit escrow automatically."
     - inputSchema: `z.object({ query: z.string().optional().describe('Search query to find a matching capability (auto-request mode)'), card_id: z.string().optional().describe('Direct card ID to request (skips search)'), skill_id: z.string().optional().describe('Specific skill within a v2.0 card'), params: z.record(z.unknown()).optional().describe('Input parameters for the capability'), max_cost: z.number().optional().default(50).describe('Maximum credits to spend') })`
   - Handler logic (mirror the CLI request command logic):
     a. If `query` is provided (auto-request mode): Create AutoRequestor with registryUrl, call `requestWithAutonomy({ query, maxCostCredits: max_cost, params })`. Return the result.
     b. If `card_id` is provided (direct mode):
        - Look up card locally first. If not found and registry is configured, fetch from remote registry.
        - Determine routing: local gateway, remote direct (has gateway_url), or relay (relay:// sentinel).
        - For remote requests with registry configured: use CreditLedger for hold/settle/release lifecycle.
        - For relay-only (no gateway_url): use relay client — relay handles credits server-side.
        - Execute and return result.
     c. If neither query nor card_id: return error "Provide either query or card_id".
   - IMPORTANT: Wrap entire handler in try/catch. On error, return `{ content: [{ type: 'text', text: JSON.stringify({ success: false, error: message }) }] }` — MCP tools should not throw.

2. Create `src/mcp/tools/conduct.ts`:
   - Export `function registerConductTool(server: McpServer, ctx: McpServerContext)`.
   - Register tool `agentbnb_conduct` with:
     - description: "Orchestrate a complex task across multiple agents on the AgentBnB network. Decomposes the task, matches sub-tasks to agents, and executes the pipeline."
     - inputSchema: `z.object({ task: z.string().describe('Natural language task description'), plan_only: z.boolean().optional().default(false).describe('If true, return execution plan without executing'), max_budget: z.number().optional().default(100).describe('Maximum credits to spend') })`
   - Handler: Call `conductAction(task, { planOnly: plan_only, maxBudget: String(max_budget) })` from `src/cli/conduct.ts`. Return the ConductResult as JSON text.

3. Create `src/mcp/tools/serve-skill.ts`:
   - Export `function registerServeSkillTool(server: McpServer, ctx: McpServerContext)`.
   - Register tool `agentbnb_serve_skill` with:
     - description: "Register as a skill provider on the AgentBnB network via relay. Connects to the registry and listens for incoming skill requests."
     - inputSchema: `z.object({ handler_url: z.string().optional().default('http://localhost:8080').describe('Local URL that handles capability execution'), skills_yaml: z.string().optional().describe('Path to skills.yaml config file') })`
   - Handler:
     a. Require ctx.config.registry to be set (return error if not).
     b. Build card data from local registry (like CLI serve does with `listCards`).
     c. Create a RelayClient with: registryUrl, owner, token, card, and an onRequest handler that uses `executeCapabilityRequest()` (import from `src/gateway/execute.ts`).
     d. Connect the relay client. Store it on ctx so it can be disconnected later.
     e. Return JSON with: success, message ("Connected to relay as provider"), owner, registry_url.
     f. NOTE: This tool starts a long-lived connection. The MCP server process stays alive, and the relay connection persists. On MCP server shutdown (SIGINT/SIGTERM), disconnect the relay.
