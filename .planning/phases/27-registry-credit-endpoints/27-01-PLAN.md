    - Test 3: Request missing X-Agent-Signature header returns 401

  <action>
Create `src/registry/identity-auth.ts` exporting a Fastify plugin `identityAuthPlugin` that:

1. Adds an `onRequest` hook that extracts three headers:
   - `X-Agent-PublicKey` (hex-encoded Ed25519 public key)
   - `X-Agent-Signature` (base64url Ed25519 signature)
   - `X-Agent-Timestamp` (ISO 8601 timestamp of request signing)

2. Verification logic:
   - If any of the 3 headers is missing, return 401 `{ error: 'Missing identity headers' }`
   - Check timestamp is within 5 minutes of current time. If expired, return 401 `{ error: 'Request expired' }`
   - Construct the signed payload: `{ method: request.method, path: request.url, timestamp: X-Agent-Timestamp, publicKey: X-Agent-PublicKey }`
   - For POST requests, also include `body: request.body` in the signed payload
   - Use `verifyEscrowReceipt(payload, signature, publicKeyBuffer)` from `src/credit/signing.ts` to verify
   - If invalid, return 401 `{ error: 'Invalid identity signature' }`
   - On success, decorate request with `request.agentPublicKey = publicKeyHex` for downstream route use

3. Register as a scoped Fastify plugin (NOT fastify-plugin) so it only applies to routes registered within the same scope.

4. Also create a helper function `signRequest(method: string, path: string, body: unknown | null, privateKey: Buffer, publicKeyHex: string): Record<string, string>` that returns the 3 headers needed to authenticate a request. This is used by tests and by RegistryCreditLedger HTTP client.

The plugin should use the existing `verifyEscrowReceipt` from `src/credit/signing.ts` — do NOT reimplement Ed25519 verification. The `verifyEscrowReceipt` function already handles canonical JSON serialization and Ed25519 verify.

Create `src/registry/identity-auth.test.ts` with a small Fastify test server that registers the plugin and a test route, then exercises all 5 behaviors above using `server.inject()`.
  </action>

  </verify>
  <done>identity-auth.ts exports identityAuthPlugin + signRequest. All 5 test cases pass. Plugin rejects unsigned/expired/bad-sig requests with 401 and passes valid ones.</done>

    - Test 9: All endpoints return 401 without auth headers

  </behavior>
  <action>
Create `src/registry/credit-routes.ts` exporting a Fastify plugin `creditRoutesPlugin` that accepts `{ creditDb: Database.Database }` options:

1. **Grant dedup table**: On plugin registration, run `CREATE TABLE IF NOT EXISTS credit_grants (public_key TEXT PRIMARY KEY, granted_at TEXT NOT NULL)` on the creditDb. This tracks which Ed25519 public keys have already received the initial 50 cr grant (per ADR-018).

2. **Register identityAuthPlugin** as a scoped plugin within creditRoutesPlugin, so all credit routes require Ed25519 auth.

3. **Routes** (all under the identity auth scope):

   **POST /api/credits/hold**
   - Body: `{ owner: string, amount: number, cardId: string }`
   - Validate all 3 fields present and amount > 0
   - Call `holdEscrow(creditDb, owner, amount, cardId)`
   - Return 200 `{ escrowId }`
   - On INSUFFICIENT_CREDITS error, return 400 `{ error: message, code: 'INSUFFICIENT_CREDITS' }`

   **POST /api/credits/settle**
   - Body: `{ escrowId: string, recipientOwner: string }`
   - Call `settleEscrow(creditDb, escrowId, recipientOwner)`
   - Return 200 `{ ok: true }`
   - On ESCROW_NOT_FOUND/ESCROW_ALREADY_SETTLED, return 400 `{ error: message, code }`

   **POST /api/credits/release**
   - Body: `{ escrowId: string }`
   - Call `releaseEscrow(creditDb, escrowId)`
   - Return 200 `{ ok: true }`
   - On ESCROW_NOT_FOUND/ESCROW_ALREADY_SETTLED, return 400 `{ error: message, code }`

   **POST /api/credits/grant**
   - Body: `{ owner: string, amount?: number }` (amount defaults to 50)
   - Read `request.agentPublicKey` (set by identity auth hook) — this is the dedup key
   - Check `credit_grants` table for this public key. If exists, return 200 `{ ok: true, granted: 0, reason: 'already_granted' }`
   - Otherwise: call `bootstrapAgent(creditDb, owner, amount ?? 50)`, insert into `credit_grants`, return 200 `{ ok: true, granted: amount ?? 50 }`

   **GET /api/credits/:owner**
   - Call `getBalance(creditDb, owner)`
   - Return 200 `{ balance }`

   **GET /api/credits/:owner/history**
   - Query param `limit` (default 20, max 100)
   - Call `getTransactions(creditDb, owner, limit)`
   - Return 200 `{ transactions, limit }`

4. **Wire into server.ts**: In `createRegistryServer`, register `creditRoutesPlugin` when `opts.creditDb` is provided. Pass `{ creditDb: opts.creditDb }` as plugin options. Register AFTER the cors and websocket plugins but BEFORE the owner routes.

5. **Update CORS**: Add `X-Agent-PublicKey`, `X-Agent-Signature`, `X-Agent-Timestamp` to the `allowedHeaders` array in the CORS registration so browsers/clients can send these headers.

Create `src/registry/credit-routes.test.ts` using the same `server.inject()` pattern as `server.test.ts`. Use `openCreditDb(':memory:')` for the credit database and `openDatabase(':memory:')` for the registry database. Use `generateKeyPair()` + `signRequest()` from identity-auth.ts to create authenticated requests. Bootstrap test agents with `bootstrapAgent()` before testing hold/settle/release.
