Enable agents to optionally publish their built-in Conductor as a paid capability card on the remote registry, alongside their SOUL.md card, when `conductor.public: true` is configured.

Output: Config toggle, relay protocol support for multi-card registration, serve command wiring, tests.
</objective>

    - Test: handleRegister with only `card` (no `cards`) still works (backward compat)

    - Test: RelayClient sends `cards` array in register message when constructed with `cards` option

  <action>
    1. **src/cli/config.ts** — Add optional `conductor?: { public: boolean }` field to `AgentBnBConfig` interface. This defaults to undefined (treated as `{ public: false }`).

    2. **src/conductor/card.ts** — Modify `buildConductorCard()` to accept an optional `owner?: string` parameter. When provided, use it as the card's `owner` field instead of `CONDUCTOR_OWNER`. This lets each agent's conductor card be attributed to the agent owner, not a shared singleton. Also make the card `id` deterministic per owner: use `crypto.createHash('sha256').update(owner).digest('hex').slice(0, 32)` formatted as UUID v4 shape, so each agent gets a unique conductor card ID. When no owner is passed, use existing behavior (CONDUCTOR_OWNER + fixed UUID).

    3. **src/relay/types.ts** — Add optional `cards` field to `RegisterMessageSchema`:
       ```typescript
       cards: z.array(z.record(z.unknown())).optional(),
       ```
       The existing `card` field stays required for backward compatibility. The `cards` array is the new multi-card path.

    4. **src/relay/websocket-relay.ts** — Update `handleRegister()`:
       - After upserting `msg.card` (existing behavior, kept for backward compat), check if `msg.cards` array exists
       - If `msg.cards` exists, iterate and call `upsertCard()` for each additional card
       - Only log `agent_joined` once (for the first/primary card)
       - Only send one `registered` ack (existing behavior)

    5. **src/relay/websocket-client.ts** — Add optional `cards?: Record<string, unknown>[]` to `RelayClientOptions`. In the `connect()` method's `on('open')` handler, include `cards` in the register message if provided:
       ```typescript
       this.send({
         type: 'register',
         owner: this.opts.owner,
         token: this.opts.token,
         card: this.opts.card,
         ...(this.opts.cards ? { cards: this.opts.cards } : {}),
       });
       ```

    6. **Tests** — Add tests in `src/conductor/card.test.ts` for `buildConductorCard(owner)` producing agent-specific card. Add a test file or extend existing relay tests to verify multi-card register message handling.
  </action>

    - buildConductorCard(owner) produces owner-specific conductor card
    - Backward compat: existing register messages (card only, no cards) still work
