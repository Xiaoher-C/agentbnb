<task type="auto">
  <name>Task 1: Create CardModal component</name>
  <files>hub/src/components/CardModal.tsx</files>
  <action>
Create a new `CardModal.tsx` component. Props: `{ card: HubCard | null; onClose: () => void }`. When `card` is `null`, render nothing (the component manages its own visibility based on the card prop).

**Overlay / backdrop:**
- Fixed position covering viewport (`fixed inset-0 z-50`)
- Background: `rgba(0,0,0,0.7)` + `backdrop-filter: blur(12px)` — use inline style for backdrop-filter or Tailwind `backdrop-blur-[12px]`
- Flex centering: `flex items-center justify-center`
- Click on backdrop (the overlay div, NOT the modal content) calls `onClose`

**Modal panel:**
- Max width: 520px (`max-w-[520px] w-full mx-4`)
- Background: `#111117`
- Border: `1px solid rgba(255,255,255,0.08)` — use `border border-white/[0.08]`
- Border radius: 20px (`rounded-modal`)
- Padding: 32px (`p-8`)
- `overflow-y: auto`, `max-h-[85vh]`
- Stop click propagation on the modal panel so clicking content does not close

**Animate in/out:**
- Use React state to track `isVisible` (for CSS transition). When `card` changes from null to non-null, set `isVisible = true`. When `onClose` is called, set `isVisible = false`, then after 150ms timeout, call the actual `onClose`.
- Animate in: opacity 0 to 1 + scale 0.96 to 1 (200ms ease-out). Use CSS classes toggled by `isVisible`.
- Animate out: opacity 1 to 0 + scale 1 to 0.96 (150ms ease-in).
- Define transition CSS using Tailwind arbitrary values or inline styles:
  ```
  transition: opacity 200ms ease-out, transform 200ms ease-out
  ```
  When closing, switch to 150ms ease-in duration.
- Simpler approach: use CSS keyframes defined in index.css if needed, or use `style` prop with `transition` and `transform`/`opacity`.

**Close triggers:**
- ESC key: `useEffect` with `keydown` listener for `Escape` key
- Backdrop click: onClick on overlay div (already handled above)

**Body scroll lock:**
- `useEffect` that adds `overflow: hidden` to `document.body.style` when modal opens, removes when it closes/unmounts. Return cleanup function.

**Modal content layout:**
1. **Header:** 48px identicon (boring-avatars Avatar `size={48}`), full name (18px semibold, `text-hub-text-primary`), owner handle (14px, `text-hub-text-tertiary`), locale if available, online status dot
2. **Description:** 14px body text, `text-hub-text-secondary`, margin-top
3. **Inputs section:** Label "Inputs" (12px uppercase, `text-hub-text-muted`, tracking-wide), list of `name: type` in `font-mono text-sm`
4. **Outputs section:** Same pattern as inputs
5. **Stats section:** Success rate, latency, credits — use `font-mono text-hub-accent` for credit numbers
6. **CLI code block:**
   - Label: "Request via CLI" (12px, `text-hub-text-muted`)
   - Code: `agentbnb request {card.id}` in `font-mono` text
   - Background: `rgba(255,255,255, 0.04)`, border `rgba(255,255,255, 0.06)`, rounded-lg, padding 12px 16px
   - Copy button: ghost style button that copies the command to clipboard. On click, show a checkmark icon for 2 seconds then revert. Use `navigator.clipboard.writeText()`. Style: small ghost button positioned at the right of the code block.
7. **Close button (optional top-right X):** Small X button at top-right corner, `text-hub-text-tertiary hover:text-hub-text-primary`, for discoverability alongside ESC/backdrop

Use `boring-avatars` Avatar (already a dependency) for the identicon. Import `formatCredits` from utils. Import `inferCategories` from categories for displaying categories in the modal.
