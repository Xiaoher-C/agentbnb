The `getActivityFeed()` function uses a single JOIN query:
- `FROM request_log r LEFT JOIN capability_cards c ON r.card_id = c.id`
- Filter: `WHERE (r.action_type IS NULL OR r.action_type = 'auto_share')` — excludes auto_request audit rows but includes auto_share (capability_shared events) and regular exchanges
- Support `since` param (ISO timestamp string, NOT SincePeriod enum): `AND r.created_at > ?`
- Support `limit` param: default 20, cap at 100 via `Math.min(limit, 100)`
- Order: `ORDER BY r.created_at DESC`
- Returns `ActivityFeedEntry[]`

2. Create `hub/src/hooks/useActivity.ts` — follows the isFirstFetch polling pattern from useAgents.ts but with key differences:
- Poll interval is 10_000ms (10s), NOT 30_000ms
- Uses `lastSeenAt` ref (string | null) to track the newest item's `created_at`
- On first fetch: `setItems(data.items)` (replace)
- On subsequent polls: `setItems(prev => [...data.items, ...prev])` (prepend only — critical for scroll position)
- Passes `?since=lastSeenAt.current` on non-first fetches
- Always passes `?limit=50` on first fetch, `?limit=20` on polls
- Derives `event_type` client-side: if `action_type === 'auto_share'` then `'capability_shared'`, else `'exchange_completed'`
- Returns `{ items: ActivityEvent[], loading: boolean, error: string | null }`
