<task type="auto">
  <name>Task 1: Wire openclaw CLI subcommand group</name>
  <files>src/cli/index.ts</files>
  <action>
    Add three subcommands under an `openclaw` command group in the existing Commander program. Follow the established pattern in src/cli/index.ts (loadConfig check, openDatabase/openCreditDb for standalone CLI access). Per RESEARCH.md Pitfall 5: these CLI commands open their own read-only DB connections (only SELECT queries) -- safe in WAL mode even while `agentbnb serve` is running.

    Add imports at the top of src/cli/index.ts:
    ```typescript
    import { publishFromSoulV2, generateHeartbeatSection, injectHeartbeatSection, getOpenClawStatus } from '../openclaw/index.js';
    import { DEFAULT_BUDGET_CONFIG } from '../credit/budget.js';  // already imported if not present
    ```

    Create the command group after existing commands:
    ```typescript
    const openclaw = program.command('openclaw').description('OpenClaw integration commands');
    ```

    **sync subcommand:**
    - Option: `--soul-path <path>` with default `'./SOUL.md'`
    - Action: loadConfig, check null. Open DB via openDatabase(config.db_path). Read soul file via readFileSync (already imported). Call publishFromSoulV2(db, content, config.owner). Print: `Published card ${card.id} with ${card.skills.length} skill(s)`. Close DB (db.close()).
    - Error handling: If soul file does not exist, print error and exit(1). If parseSoulMdV2 throws (no H2 sections), print the error message and exit(1).

    **status subcommand:**
    - No options.
    - Action: loadConfig, check null. Open both DBs. Call getOpenClawStatus(config, db, creditDb). Print formatted output:
      ```
      AgentBnB OpenClaw Status
      Owner: {owner}
      Gateway: {gateway_url}
      Tier 1 (auto): < {tier.tier1_max_credits} credits
      Tier 2 (notify): {tier.tier1_max_credits}-{tier.tier2_max_credits} credits
      Tier 3 (ask): > {tier.tier2_max_credits} credits
      Balance: {balance} credits
      Reserve: {reserve} credits
      Skills: {skills.length}
      ```
      Then for each skill: `  - {id}: {name} (idle: {idle_rate ?? 'N/A'}, online: {online})`
    - Close both DBs.

    **rules subcommand:**
    - Option: `--inject <path>` (optional) for auto-patching a HEARTBEAT.md file
    - Action: loadConfig, check null. Read autonomy config (config.autonomy ?? DEFAULT_AUTONOMY_CONFIG) and budget config (config.budget ?? DEFAULT_BUDGET_CONFIG). Call generateHeartbeatSection(autonomy, budget). If --inject provided, call injectHeartbeatSection(opts.inject, section) and print `Injected AgentBnB rules into {opts.inject}`. If no --inject, print the section to stdout.
  </action>
