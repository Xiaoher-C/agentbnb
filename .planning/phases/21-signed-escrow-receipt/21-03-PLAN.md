<objective>
Implement the settlement protocol for independent credit resolution on both sides of a cross-machine transaction.

Purpose: In P2P mode, there is no shared DB. The provider and requester each update their own local SQLite independently. The provider records earnings; the requester settles or releases the escrow it held. This plan creates the settlement functions that Plan 02 (gateway) and Plan 04 (integration tests) will use.

Output: settlement.ts with provider/requester settlement functions, updated ledger.ts with recordEarning.

      ```typescript
      export function recordEarning(
        db: Database.Database,
        owner: string,
        amount: number,
        cardId: string,
        receiptNonce: string,
      ): void {
        const now = new Date().toISOString();
        db.transaction(() => {
          // Idempotency: check if this nonce was already recorded
          const existing = db.prepare(
            "SELECT id FROM credit_transactions WHERE reference_id = ? AND reason = 'remote_earning'"
          ).get(receiptNonce);
          if (existing) return; // Already recorded — skip

          // Ensure balance row exists
          db.prepare('INSERT OR IGNORE INTO credit_balances (owner, balance, updated_at) VALUES (?, 0, ?)').run(owner, now);
          // Credit the earnings
          db.prepare('UPDATE credit_balances SET balance = balance + ?, updated_at = ? WHERE owner = ?').run(amount, now, owner);
          // Log transaction with receipt nonce as reference_id
          db.prepare('INSERT INTO credit_transactions (id, owner, amount, reason, reference_id, created_at) VALUES (?, ?, ?, ?, ?, ?)')
            .run(randomUUID(), owner, amount, 'remote_earning', receiptNonce, now);
        })();
      }
      ```
