    - Receipt amount < creditsNeeded returns JSON-RPC error -32603 "Insufficient escrow amount"
    - Expired receipt (timestamp > 5 minutes old) returns JSON-RPC error -32603 "Escrow receipt expired"
    - Valid receipt: provider proceeds with execution, no local escrow hold (credits are on requester's side)
    - On success with remote receipt: response includes `receipt_settled: true` flag so requester knows to settle their local escrow
    - On failure with remote receipt: response includes `receipt_released: true` flag so requester knows to release their local escrow
  </behavior>
  <action>
    Modify `src/gateway/server.ts` POST /rpc handler (around lines 175-194, the "Check balance and hold escrow" section):

    1. Import `verifyEscrowReceipt` from `../credit/signing.js` and `EscrowReceipt` from `../types/index.js`.

    2. Replace the credit check block with a branching flow:

    ```typescript
    const receipt = params.escrow_receipt as EscrowReceipt | undefined;
    let escrowId: string | null = null;
    let isRemoteEscrow = false;

    if (receipt) {
      // Remote P2P path: verify signed receipt
      const { signature, ...receiptData } = receipt;
      const publicKeyBuf = Buffer.from(receipt.requester_public_key, 'hex');
      const valid = verifyEscrowReceipt(receiptData as Record<string, unknown>, signature, publicKeyBuf);
      if (!valid) {
        return reply.send({ jsonrpc: '2.0', id, error: { code: -32603, message: 'Invalid escrow receipt signature' } });
      }
      if (receipt.amount < creditsNeeded) {
        return reply.send({ jsonrpc: '2.0', id, error: { code: -32603, message: 'Insufficient escrow amount' } });
      }
      // Check receipt freshness (5 minute window)
      const receiptAge = Date.now() - new Date(receipt.timestamp).getTime();
      if (receiptAge > 5 * 60 * 1000) {
        return reply.send({ jsonrpc: '2.0', id, error: { code: -32603, message: 'Escrow receipt expired' } });
      }
      isRemoteEscrow = true;
      // No local escrow hold — credits are committed on requester's machine
    } else {
      // Local/same-machine path: existing behavior
      try {
        const balance = getBalance(creditDb, requester);
        if (balance < creditsNeeded) { ... }
        escrowId = holdEscrow(creditDb, requester, creditsNeeded, cardId);
      } catch (err) { ... }
    }
    ```

    3. For settlement after successful execution:
    - If `isRemoteEscrow`: Do NOT call settleEscrow (no local escrow exists). Do NOT call recordEarning (that function is created by Plan 21-03 and will be wired in Plan 21-04's integration). Instead, include `receipt_settled: true` and the receipt nonce in the success response so the settlement layer (Plan 21-03) can handle earning recording when it is wired in.
    - If local escrow (existing path): keep existing `settleEscrow(creditDb, escrowId!, card.owner)` call unchanged.

    4. For failure after execution:
    - If `isRemoteEscrow`: Include `receipt_released: true` in the error response so the requester knows to release their local escrow. The provider takes no local action (no escrow was held locally).
    - If local escrow: keep existing `releaseEscrow(creditDb, escrowId!)` call unchanged.

    IMPORTANT: This plan does NOT modify `src/credit/ledger.ts`. The `recordEarning` function and `remote_earning` reason type are created by Plan 21-03 (settlement protocol). This plan only handles receipt verification and the gateway request/response flow.

    Write tests in `src/gateway/server.test.ts` that:

    - Send /rpc with expired receipt -> gets expired error
    - Send /rpc without receipt -> falls back to local DB check (backward compat)
