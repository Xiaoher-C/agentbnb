Add two functions to `src/registry/store.ts`. Both use the direct SQL read/mutate/write pattern -- DO NOT use `updateCard()` which rejects v2.0 shapes via v1.0 Zod validation.

**`updateSkillAvailability(db, cardId, skillId, online)`:**
1. `SELECT data FROM capability_cards WHERE id = ?`
2. `JSON.parse(row.data)` -> find skill by `skills.find(s => s.id === skillId)`
3. Set `skill.availability = { ...(skill.availability ?? {}), online }`
4. `UPDATE capability_cards SET data = ?, updated_at = ? WHERE id = ?`
5. No-op if card or skill not found (return silently).

**`updateSkillIdleRate(db, cardId, skillId, idleRate)`:**
1. Same read pattern as above.
2. Set `skill._internal = { ...(skill._internal ?? {}), idle_rate: idleRate, idle_rate_computed_at: new Date().toISOString() }`
3. Same write pattern. Preserves any other keys already in `_internal`.
4. No-op if card or skill not found.

Write tests FIRST in `src/registry/store.test.ts` (append to existing file). Create an in-memory DB, insert a v2.0 card with 2 skills using direct SQL (INSERT INTO capability_cards with JSON.stringify of a v2.0 card shape including `spec_version: '2.0'` and `skills: [{ id: 'skill-a', ... }, { id: 'skill-b', ... }]`).
