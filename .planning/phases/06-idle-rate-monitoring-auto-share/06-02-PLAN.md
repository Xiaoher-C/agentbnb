Create `src/autonomy/idle-monitor.ts` following the research pattern exactly:

```typescript
export interface IdleMonitorOptions {
  owner: string;
  db: Database.Database;
  pollIntervalMs?: number;      // default: 60_000
  idleThreshold?: number;       // default: 0.70
  autonomyConfig: AutonomyConfig;
}

export class IdleMonitor {
  private readonly job: Cron;
  constructor(opts: IdleMonitorOptions) { ... }
  start(): Cron { ... }
  getJob(): Cron { ... }
  async poll(): Promise<void> { ... }
}
```

**poll() logic:**
1. `listCards(db, owner)` to get all cards owned by this agent
2. For each card, cast to unknown and check for `skills[]` array (skip v1.0 cards without it)
3. For each skill: compute `capacity = skill.metadata?.capacity?.calls_per_hour ?? 60`
4. Call `getSkillRequestCount(db, skill.id, 60 * 60 * 1000)` for the 60-minute window
5. Compute `idleRate = Math.max(0, 1 - (count / capacity))` -- clamp to prevent negative
6. Call `updateSkillIdleRate(db, card.id, skill.id, idleRate)` to persist to _internal
7. If `idleRate >= threshold && !skill.availability?.online`:
   - Call `getAutonomyTier(0, autonomyConfig)` -- auto-share costs 0 credits
   - Tier 1: `updateSkillAvailability(db, card.id, skill.id, true)` + `insertAuditEvent(db, { type: 'auto_share', ... })`
   - Tier 2: same flip + `insertAuditEvent(db, { type: 'auto_share_notify', ... })`
   - Tier 3: `insertAuditEvent(db, { type: 'auto_share_pending', ... })` only -- do NOT flip

**Cron setup:** Use `0 * * * * *` (6-field croner expression, fires at second 0 of every minute). Construct with `{ paused: true }`. `start()` calls `this.job.resume()` and returns the job. Use `void this.poll()` in callback (fire-and-forget -- croner callbacks are not async-aware).

**Tests:** Use in-memory DB with `openDatabase(':memory:')` + `createRequestLogTable(db)`. Insert v2.0 cards with skills[] directly via SQL. Call `monitor.poll()` directly (no need to test croner timing). Mock time by inserting request_log rows with controlled `created_at` timestamps.

For Tier 1/2/3 tests, construct IdleMonitor with different `autonomyConfig` values:
- Tier 1: `{ tier1_max_credits: 10, tier2_max_credits: 50 }` (0 < 10 -> Tier 1)
- Tier 2: `{ tier1_max_credits: 0, tier2_max_credits: 50 }` (0 >= 0, 0 < 50 -> actually Tier 2... but note: getAutonomyTier uses strict less-than. With tier1=0: amount 0 is NOT < 0, so not Tier 1. With tier2=50: amount 0 IS < 50, so Tier 2. Correct.)
- Tier 3: DEFAULT_AUTONOMY_CONFIG `{ tier1_max_credits: 0, tier2_max_credits: 0 }` (0 >= 0, 0 >= 0 -> Tier 3)

IMPORTANT: Auto-share costs 0 credits. With `getAutonomyTier(0, config)`:
- Tier 1 requires `0 < tier1_max_credits` -> need tier1_max_credits > 0 (e.g., 10)
- Tier 2 requires `0 >= tier1_max_credits AND 0 < tier2_max_credits` -> need tier1=0, tier2 > 0 (e.g., 50)
- Tier 3 requires `0 >= tier2_max_credits` -> both at 0 (default)
