Output: Fee calculation function, relay integration that detects Conductor responses and settles the fee, and tests.

       ```typescript
       export function calculateConductorFee(totalSubTaskCost: number): number {
         if (totalSubTaskCost <= 0) return 0;
         const fee = Math.ceil(totalSubTaskCost * 0.1);
         return Math.max(1, Math.min(20, fee));
       }
       ```
       Use `Math.ceil` so fractional credits round up (e.g., 15 * 0.1 = 1.5 -> 2 cr).

    2. **Add tests for `calculateConductorFee` to `src/relay/relay-credit.test.ts`:**

       - 100 cost -> 10 fee
       - 200 cost -> 20 fee (max)
       - 300 cost -> 20 fee (max cap)

    3. **Modify `handleRelayResponse` in `websocket-relay.ts`** to detect Conductor responses and settle the fee:
       - After settling the main escrow (from Plan 01), check if the response result looks like a Conductor orchestration result: `typeof msg.result === 'object' && msg.result !== null && 'total_credits' in msg.result`.
       - If so, extract `total_credits` from `msg.result` as a number.
       - Call `calculateConductorFee(totalCredits)`.
       - If fee > 0, attempt to hold+settle the fee from the original requester (`pending.originOwner`) to the conductor agent (`pending.targetOwner`):
         - `const feeEscrowId = holdForRelay(creditDb, pending.originOwner, fee, msg.id)` — use msg.id as cardId for the fee escrow (it's just a reference).
         - `settleForRelay(creditDb, feeEscrowId, pending.targetOwner!)`.
       - Wrap in try/catch — if the requester has insufficient credits for the fee, log it but do not fail the response. The main capability was already settled successfully. The fee is best-effort (the requester might have spent their remaining credits on the sub-tasks).
       - Add the fee amount to the response forwarded to the requester so they know what was charged: add `conductor_fee` field to the forwarded response message.

    4. **Add integration test** in relay-credit.test.ts: test that calculateConductorFee + holdForRelay + settleForRelay work together — hold fee from requester, settle to conductor, verify balances.
