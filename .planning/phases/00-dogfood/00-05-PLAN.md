    1. Create `src/skills/publish-capability.ts`:
       - `parseSoulMd(content: string)`: Parses SOUL.md markdown content. Extracts: agent name (first H1), description (first paragraph), capabilities (H2 sections). Returns partial CapabilityCard data. Defaults to level 2.
       - `publishFromSoul(db, soulContent, owner)`: Calls parseSoulMd, fills in remaining fields (id via randomUUID, pricing defaults to 10 credits/call, availability online:true), validates with CapabilityCardSchema, inserts into registry. Returns the created card.

    2. Create `src/skills/handle-request.ts`:
       - `createRequestHandler(handlers)`: Takes a map of card-id to async handler functions `{ [cardId]: (params) => Promise<result> }`. Returns an HTTP handler function compatible with gateway's handlerUrl pattern (accepts POST with JSON body `{ card_id, params }`, returns `{ result }`).
       - Exposes as a simple Fastify server factory for local handler.

    3. Create `src/skills/integration.test.ts` with end-to-end test:
       - Set up: In-memory DBs for registry and credit. Bootstrap Agent A and Agent B with 100 credits each.
       - Agent B publishes a capability card (e.g., "Text Summarizer" — takes text input, returns shortened text).
       - Agent B starts a local handler server that actually processes requests.
       - Agent B starts a gateway server pointing to the local handler.
       - Agent A discovers "summarizer" via search.
       - Agent A requests execution via gateway client.
       - Assert: Agent A receives summarized text result.
       - Assert: Agent A's balance decreased by card pricing.
       - Assert: Agent B's balance increased by card pricing.
       - Teardown: Close all servers.

    SOUL.md parsing should be defensive — use regex for H1/H2, not a full markdown parser (no new dependency). Flag unknown sections rather than crash. Per RESEARCH.md Open Question 1: default to L2, flag for manual review.
