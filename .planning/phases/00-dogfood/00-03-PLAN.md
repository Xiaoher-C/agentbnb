  <action>
    1. Create `src/gateway/auth.ts`:
       - Export a Fastify plugin `authPlugin` that checks `Authorization: Bearer <token>` header against a configured token list. Skip auth for `/health`. Return 401 JSON-RPC error on failure.

    2. Create `src/gateway/server.ts` with (Pattern 5 from RESEARCH.md):
       - `createGatewayServer(opts)`: Takes `{ port, registryDb, creditDb, tokens, handlerUrl, timeoutMs }`. Creates Fastify instance with logger.
       - GET `/health` — returns `{ status: 'ok', version, uptime }`.
       - POST `/rpc` — JSON-RPC 2.0 endpoint. Validates body structure. Dispatches methods:
         - `capability.execute`: Looks up card by `params.card_id` in registry. Holds escrow for `pricing.credits_per_call`. POSTs to `handlerUrl` with card inputs. On success: settles escrow, returns result. On failure/timeout: releases escrow, returns error.
       - Timeout: Use AbortController with configurable timeout (default 30s).
       - Request logging via Fastify logger.

    3. Create `src/gateway/server.test.ts`: Use `server.inject()` for testing (no actual port needed). Mock the handler URL with a simple Fastify server in tests.
  </action>
