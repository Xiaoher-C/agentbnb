<task type="auto">
  <name>Task 2: Wire connect, peers, --announce, --local into CLI</name>
  <files>src/cli/index.ts, src/cli/config.ts</files>
  <action>
    1. Add `agentbnb connect <name> <url> <token>` command to `src/cli/index.ts`:
       - Validates config exists (init required)
       - Calls `savePeer({ name, url, token, added_at: new Date().toISOString() })`
       - Prints success: "Connected to peer: {name} at {url}"
       - Supports `--json` flag

    2. Add `agentbnb peers` command:
       - Lists all registered peers (name, url, added_at)
       - Table format by default, `--json` for JSON output
       - "No peers registered. Use `agentbnb connect` to add one." when empty

    3. Add `agentbnb peers remove <name>` subcommand:
       - Calls `removePeer(name)`
       - Prints success or "Peer not found: {name}"

    4. Update `agentbnb request <card-id>` command:
       - Add `--peer <name>` option
       - When `--peer` is provided: look up peer via `findPeer(name)`, use peer.url as gatewayUrl and peer.token as token (instead of config.gateway_url / config.token)
       - When `--peer` is NOT provided: keep existing behavior (use config.gateway_url + config.token)
       - Error if `--peer` name not found: "Peer not found: {name}. Run `agentbnb peers` to see registered peers."

    5. Add `--announce` flag to `agentbnb serve`:
       - When `--announce` is passed, call `announceGateway(config.owner, port)` after server starts
       - Add `await stopAnnouncement()` to the gracefulShutdown handler (before server.close)
       - Print "Announcing on local network via mDNS" when active

    6. Add `--local` flag to `agentbnb discover`:
       - When `--local` is passed, call `discoverLocalAgents()` instead of querying local registry
       - Wait 3 seconds for responses, then stop browsing and print results
       - Print table of discovered agents: Name, URL, Owner
       - Can be combined with existing registry query (but --local overrides registry search)

    7. Update `src/cli/config.ts` — no structural changes needed, but update the `AgentBnBConfig` JSDoc to note that `gateway_url` should use LAN IP for multi-machine scenarios (documentation only, actual LAN IP detection is Plan 04).
