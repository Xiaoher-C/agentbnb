<task type="auto">
  <name>Task 2: Create screenshot script and capture Hub screenshot</name>
  <files>scripts/take-screenshot.mjs, docs/hub-screenshot.png</files>
  <action>
Step 1: Create directory `scripts/` if it does not exist.

Step 2: Create `scripts/take-screenshot.mjs` — a Node.js ESM script that:

1. Spawns `npx vite preview --port 4173` from the `hub/` directory (the Hub must be pre-built with `pnpm build:hub`)
2. Waits for the server to be ready (poll `http://localhost:4173/hub/` with fetch in a loop, max 10 seconds)
3. Launches Playwright Chromium headless via `import { chromium } from 'playwright'`
4. Sets viewport to `{ width: 1280, height: 800 }`
5. Navigates to `http://localhost:4173/hub/` with `{ waitUntil: 'networkidle' }`
6. Waits an additional 1 second for any CSS animations/transitions to settle
7. Takes a screenshot saved to `docs/hub-screenshot.png`
8. Closes browser, kills the preview server process
9. Logs the file size to confirm success

Key implementation details:
- Use `import { spawn } from 'child_process'` for the preview server
- Use `import { chromium } from 'playwright'` (globally installed at /opt/homebrew/bin/playwright, v1.58.0)
- The Hub base path is `/hub/` (from `hub/vite.config.ts` `base: '/hub/'`). The correct URL is `http://localhost:4173/hub/`, NOT `http://localhost:4173/`
- The preview server will NOT have a backend, so API calls (`/cards`, `/api/*`) will fail. The Hub renders a graceful empty/fallback state which is fine — the requirement (README-04) says "real screenshot," not "populated screenshot"
- Add `process.exit(0)` at the end to ensure clean exit

Step 3: Run the script to produce the actual screenshot:
```bash
cd /Users/xiaoher/Documents/GitHub/agentbnb
pnpm build:hub
node scripts/take-screenshot.mjs
```

Step 4: Verify `docs/hub-screenshot.png` is non-zero bytes.

If the screenshot shows a completely blank page, check that the URL used is `http://localhost:4173/hub/` (with the /hub/ prefix) and that `pnpm build:hub` completed successfully.
  </action>
