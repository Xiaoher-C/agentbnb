## Accomplishments
- Added `_internal: z.record(z.unknown()).optional()` to CapabilityCardSchema — private per-card metadata that can be stored but never transmitted
- Added `free_tier: z.number().nonnegative().optional()` to pricing — enables freemium capability listings
- Added `stripInternal()` helper in registry server, applied to both GET /cards (paginated items) and GET /cards/:id
- Stripped `_internal` from CLI discover output before both `--json` and table paths
- Added `free_tier?: number` to HubCard.pricing interface
- Rendered "N free/mo" badge in CapabilityCard stats row when `free_tier > 0`
- 8 new tests: 4 schema validation tests, 2 server stripping tests, 2 Hub badge tests

## Decisions Made
- `_internal` stored in SQLite via schema (Zod no longer strips it as unknown) but stripped at the API boundary via `stripInternal()`. Enables private metadata without adding a separate DB column.
- `free_tier: 0` is valid (nonnegative constraint). Badge only renders for `> 0` so zero effectively disables the badge without being an invalid value.
- Hub badge guard uses `!== undefined && > 0` — explicit undefined check required because 0 is falsy but valid.
- CLI stripping applied once after both `searchCards` and `filterCards` paths, before any output — covers JSON and table formats with a single strip line.
