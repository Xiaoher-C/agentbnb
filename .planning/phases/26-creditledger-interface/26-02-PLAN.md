  <name>Task 1: Implement RegistryCreditLedger with dual-mode operation</name>
  <files>src/credit/registry-credit-ledger.ts, src/credit/registry-credit-ledger.test.ts</files>
  <behavior>
    - Direct DB mode: when constructed with { db }, delegates to existing ledger.ts/escrow.ts functions (same as LocalCreditLedger but for Registry server process)
    - HTTP client mode: when constructed with { registryUrl, ownerPublicKey }, makes HTTP calls to /api/credits/* endpoints
    - HTTP hold → POST /api/credits/hold { owner, amount, cardId } → returns { escrowId }
    - HTTP settle → POST /api/credits/settle { escrowId, recipientOwner }
    - HTTP release → POST /api/credits/release { escrowId }
    - HTTP getBalance → GET /api/credits/:owner → returns { balance }
    - HTTP getHistory → GET /api/credits/:owner/history?limit=N → returns { transactions }
    - HTTP grant → POST /api/credits/grant { owner, amount }
    - HTTP errors (non-2xx) are converted to AgentBnBError with appropriate codes
    - HTTP mode includes X-Agent-Owner and X-Agent-PublicKey headers for auth (Phase 27 will validate these)
  </behavior>
  <action>
    1. Create `src/credit/registry-credit-ledger.ts`:
       - Define `RegistryCreditLedgerConfig` type with two shapes:
         - Direct DB: `{ mode: 'direct'; db: Database.Database }`
         - HTTP client: `{ mode: 'http'; registryUrl: string; ownerPublicKey: string }`
       - Class `RegistryCreditLedger` implements `CreditLedger`
       - Constructor takes `RegistryCreditLedgerConfig`
       - For direct mode: delegate to ledger.ts/escrow.ts functions (identical pattern to LocalCreditLedger)
       - For HTTP mode: use native `fetch` to call Registry API endpoints
         - All requests include headers: `{ 'Content-Type': 'application/json', 'X-Agent-Owner': owner, 'X-Agent-PublicKey': ownerPublicKey }`
         - Parse response JSON, throw `AgentBnBError` on non-2xx with code from response body or 'REGISTRY_ERROR'
         - Use 10s timeout via AbortController for each request
       - Export the class

         - Test settle sends POST to /api/credits/settle

         - Test getBalance sends GET to /api/credits/{owner}
         - Test getHistory sends GET to /api/credits/{owner}/history?limit=N
         - Test grant sends POST to /api/credits/grant
         - Test HTTP error handling: non-2xx → AgentBnBError
         - Test network error (fetch throws) → AgentBnBError with REGISTRY_UNREACHABLE code
