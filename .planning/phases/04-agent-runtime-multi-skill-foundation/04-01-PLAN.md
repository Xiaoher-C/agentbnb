  <action>
    Create `src/runtime/agent-runtime.ts` with the `AgentRuntime` class following the research pattern:

    1. **Constructor** accepts `RuntimeOptions { registryDbPath, creditDbPath, owner, orphanedEscrowAgeMinutes? }`. Opens both databases with `new Database(path)`, sets WAL mode, foreign_keys=ON, busy_timeout=5000 on each. Stores `draining = false`.

    2. **registerJob(job: Cron)** pushes the Cron instance to `this.jobs[]`.

    3. **start()** calls `this.recoverOrphanedEscrows()`. This method queries `credit_escrow` for rows with `status = 'held' AND created_at < cutoff` (cutoff = now minus orphanedEscrowAgeMinutes, default 10). For each orphaned row, calls `releaseEscrow(this.creditDb, row.id)` in a try/catch (silent on failure — escrow may have settled between query and release).

    4. **shutdown()** sets `this.draining = true`, calls `job.stop()` on all registered jobs, closes `registryDb` and `creditDb`. Guard against double-close by checking if already draining.

    5. **get isDraining()** returns `this.draining`.

    Export `AgentRuntime` class and `RuntimeOptions` interface.

    For tests: Use in-memory SQLite databases. The test must manually create the `credit_escrow` and `credit_balances` tables (copy schema from `src/credit/ledger.ts`). Use `croner` to create a test Cron job that can be verified as stopped.

    Install new dependencies first: `pnpm add croner typed-emitter`
  </action>
