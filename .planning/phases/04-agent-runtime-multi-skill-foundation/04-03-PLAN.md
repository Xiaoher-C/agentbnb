    **Part A: Update `src/gateway/server.ts`**

    1. Import `CapabilityCardV2` and `Skill` types from `../types/index.js`.

    2. In the POST /rpc handler, after extracting `cardId`, also extract `skillId`:
       ```typescript
       const skillId = params.skill_id as string | undefined;
       ```

    3. After getting the card from registry, determine the skill and pricing:
       ```typescript
       const card = getCard(registryDb, cardId);
       // ... null check ...

       // Resolve skill: if card is v2.0 with skills[], look up by skill_id
       let creditsNeeded: number;
       let skillName: string;
       let resolvedSkillId: string | undefined;

       if ('skills' in card && Array.isArray((card as any).skills)) {
         const v2card = card as unknown as CapabilityCardV2;
         const skill = skillId
           ? v2card.skills.find(s => s.id === skillId)
           : v2card.skills[0]; // fallback to first skill

         if (!skill) {
           return reply.send({
             jsonrpc: '2.0', id,
             error: { code: -32602, message: `Skill not found: ${skillId}` },
           });
         }

         creditsNeeded = skill.pricing.credits_per_call;
         skillName = skill.name;
         resolvedSkillId = skill.id;
       } else {
         // v1.0 card — use card-level pricing
         creditsNeeded = card.pricing.credits_per_call;
         skillName = card.name;
       }
       ```

    4. Update the handler URL POST body to include `skill_id`:
       ```typescript
       body: JSON.stringify({ card_id: cardId, skill_id: resolvedSkillId, params }),
       ```

    5. Update all `insertRequestLog` calls to include `skill_id: resolvedSkillId`:
       ```typescript
       insertRequestLog(registryDb, {
         id: randomUUID(),
         card_id: cardId,
         card_name: skillName,
         skill_id: resolvedSkillId,
         requester,
         status: 'success',
         // ...
       });
       ```

    6. Note: `getCard()` currently returns `CapabilityCard | null`. After migration, all cards are v2.0 in the DB, but the return type is still `CapabilityCard`. The function should be updated to return `AnyCard | null` or we cast. For minimal changes, cast to `unknown` and check for `skills` property. Prefer: keep getCard as-is but cast the result in server.ts since store.ts is owned by Plan 02.
