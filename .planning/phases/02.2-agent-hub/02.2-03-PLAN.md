  <files>src/registry/server.ts, src/cli/index.ts, package.json, hub/vite.config.ts</files>
  <action>
1. **Install @fastify/static** as a dependency in the ROOT package.json (not hub/):
   - Run `pnpm add @fastify/static` in the project root
   - This is the standard Fastify plugin for serving static files

2. **Update src/registry/server.ts** -- Add static file serving for the hub:

   After the existing CORS registration, register @fastify/static:
   ```typescript
   import fastifyStatic from '@fastify/static';
   import { join, dirname } from 'node:path';
   import { fileURLToPath } from 'node:url';
   import { existsSync } from 'node:fs';

   // In createRegistryServer(), after cors registration:
   // Resolve hub dist directory relative to this file's location
   // In production (built): dist/registry/server.js -> need to go up to project root -> hub/dist/
   // Use a heuristic: check multiple possible locations
   const __filename = fileURLToPath(import.meta.url);
   const __dirname = dirname(__filename);

   // Try to find hub/dist from project root or relative to dist/
   const hubDistCandidates = [
     join(__dirname, '../../hub/dist'),      // When running from dist/registry/server.js
     join(__dirname, '../../../hub/dist'),    // Fallback
   ];
   const hubDistDir = hubDistCandidates.find(p => existsSync(p));

   if (hubDistDir) {
     void server.register(fastifyStatic, {
       root: hubDistDir,
       prefix: '/hub/',
       decorateReply: false,  // Avoid decorator collision if used elsewhere
     });

     // Serve index.html for /hub (without trailing slash)
     server.get('/hub', async (_request, reply) => {
       return reply.redirect('/hub/');
     });
   }
   ```

   IMPORTANT: The static serving should be optional -- if hub/dist/ doesn't exist (e.g. hub not built yet), the server still starts fine without the hub. This is critical for backward compatibility.

   IMPORTANT: Do NOT break existing tests. The existing registry server tests inject requests and expect specific responses. Since those tests won't have a hub/dist/ directory, the static plugin simply won't register.

3. **Update hub/vite.config.ts** -- Ensure build output and base path are correct:


4. **Update root package.json** -- Add convenience scripts:
   - Add `"build:hub": "cd hub && pnpm install && pnpm build"` to scripts
   - Add `"build:all": "pnpm build && pnpm build:hub"` to build everything
