### Human Verification Required

The automated verification is complete and all 14 truths are satisfied by code analysis. However, the following items require human browser testing because they involve live server behavior, visual rendering, and user interactions that cannot be verified statically:

#### 1. Hub page loads at /hub with correct visual design

**Test:** Run `pnpm build:all && node dist/cli/index.js serve --registry-port 7701`, then open `http://localhost:7701/hub` in a browser.

**Expected:** Dark slate-900 background, "AgentBnB Hub" heading, stats bar showing counts (zeros acceptable), search/filter controls, and either a card grid or the "No agents online yet" empty state with `npx agentbnb init` command.

**Why human:** Static file serving via `@fastify/static` requires a running server; Vite `base: '/hub/'` asset URL resolution needs a real browser request chain.

#### 2. Card expand-in-place behavior

**Test:** If cards are present, click one card. Then click it again.

**Expected:** First click expands the card in-place (not a modal) showing description, Inputs/Outputs, latency, and the `agentbnb request <id>` CLI snippet. Second click collapses it back.

**Why human:** The `expandedId` state toggle and CSS transition rendering require browser DOM interaction.

#### 3. Responsive grid layout

**Test:** Resize the browser window from ~1400px wide to ~400px.

**Expected:** Card grid reflows from 4 columns to 2 columns to 1 column naturally. No horizontal scrollbar.

**Why human:** CSS `auto-fit` grid behavior depends on actual viewport width and can only be verified visually.

#### 4. Live search and filter

**Test:** Type text in the search input. Select "L1 Atomic" from the level dropdown. Check "Online only".

**Expected:** Each filter change triggers a new fetch to `/cards?q=...&level=1&online=true` and cards update within a second.

**Why human:** Network round-trip timing, URL construction in a real browser, and React re-render after fetch cannot be verified by static analysis.
