    - Test: `agentbnb init --owner test --no-detect` creates config but publishes zero cards, stdout does NOT contain "Detected"
    - Test: `agentbnb init --owner test --yes` with OPENAI_API_KEY in env publishes a draft card and stdout contains "OpenAI"
    - Test: `agentbnb init --owner test --yes` without any known env vars publishes zero cards, stdout says "No API keys detected"
    - Test: `agentbnb init --owner test --no-detect --json` JSON output does NOT contain "detected" or "draft" keys
    - Test: existing init behavior preserved (config.json created, credits bootstrapped, token generated)
    - Test: `agentbnb init --owner test` (no flags, non-TTY) with OPENAI_API_KEY set: detects key, prints detection summary, does NOT publish card, stdout contains skip notice explaining --yes flag
  </behavior>
  <action>
    Modify the `init` command in `src/cli/index.ts`:

    1. Add two new options to the init command:
       - `.option('--yes', 'Auto-confirm all draft cards (non-interactive)')`
       - `.option('--no-detect', 'Skip API key detection')`

    2. After existing config save + credit bootstrap, add onboarding flow with this control structure:

       **If `opts.noDetect`: skip steps 2a-2g entirely, go to step 3.** This is the primary guard -- `--no-detect` means zero detection work happens.

       a. Call `detectApiKeys(KNOWN_API_KEYS)` to get detected API key names
       b. Call `detectOpenPorts([7700, 7701, 8080, 3000, 8000, 11434])` to find running services
       c. Print detection summary to stdout (e.g., "Detected 3 API keys: OPENAI_API_KEY, ANTHROPIC_API_KEY, ELEVENLABS_API_KEY")
       d. Print open ports if any found (e.g., "Found services on ports: 8080, 11434")
       e. For each detected API key, call `buildDraftCard(apiKey, owner)` to generate draft
       f. Determine publish mode:
          - If `opts.yes`: auto-publish all draft cards via `insertCard(db, card)`
          - If `process.stdout.isTTY` (and no `--yes`): use readline confirm prompt ("Publish this card? [y/N]") for each draft card
          - If non-TTY (and no `--yes`): skip publishing entirely and print notice: "Non-interactive environment detected. Re-run with --yes to auto-publish draft cards."
       g. If no API keys detected, print "No API keys detected. You can manually publish cards with `agentbnb publish`."

    3. For `--json` output mode: include `detected_keys: string[]` and `published_cards: Array<{id, name}>` in the JSON response.

    4. Import `createInterface` from `node:readline` for the confirm prompt. Implement a local `confirm(question: string): Promise<boolean>` helper that creates a readline interface, asks the question with `[y/N]`, and resolves with the answer. Always close the readline interface in a finally block.

    Add integration tests to `src/cli/index.test.ts` using the existing `runCli()` helper pattern with AGENTBNB_DIR isolation. Set env vars in the execSync call to simulate detected keys. The non-TTY test is naturally covered because execSync spawns a child process which has no TTY.
  </action>
  <verify>
    <automated>pnpm vitest run src/cli/index.test.ts --reporter=verbose</automated>
  </verify>

    - Non-TTY without --yes: detection runs, summary printed, publishing skipped with notice
    - Existing init behavior (config, credits, token) unchanged
    - All new and existing integration tests pass
  </done>
