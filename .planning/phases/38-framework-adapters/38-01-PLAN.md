<interfaces>
<!-- Registry HTTP API endpoints the adapters will call -->

Public (no auth):
  GET  /cards?q=<query>&limit=N&offset=N  — Search/list capability cards
  GET  /cards/:id                          — Get single card by ID
  GET  /api/pricing?q=<query>              — Get market pricing stats
  GET  /api/agents                         — List agent profiles
  GET  /api/agents/:owner                  — Get agent profile
  GET  /health                             — Health check

Ed25519 authenticated (3 headers required):
  POST /api/credits/hold    { amount, cardId, skillId? }  — Hold escrow
  POST /api/credits/settle  { escrowId }                  — Settle escrow to provider
  POST /api/credits/release { escrowId }                  — Release escrow (refund)
  POST /api/credits/grant   {}                            — Request initial 50 cr grant
  GET  /api/credits/:owner                                — Get balance
  GET  /api/credits/:owner/history                        — Get transaction history

Ed25519 Auth Header Protocol (from identity-auth.ts):
  X-Agent-PublicKey: hex-encoded Ed25519 public key (DER/SPKI format)
  X-Agent-Signature: base64url Ed25519 signature over canonical JSON of { method, path, publicKey, timestamp }
  X-Agent-Timestamp: ISO 8601 timestamp (must be within 5 minutes of server time)

Signature payload is canonical JSON (sorted keys) of:
  { method: "POST", path: "/api/credits/hold", publicKey: "<hex>", timestamp: "<iso>" }

Key format: Ed25519 keypair stored as DER-encoded buffers
  - Private key: PKCS8 DER format
  - Public key: SPKI DER format

GPT Actions export:
  GET /api/openapi/gpt-actions?server_url=<url>  — Returns filtered OpenAPI 3.0 spec
</interfaces>

Create three standalone Python adapter files. Each is self-contained (no shared package, no setup.py). All three share the same Ed25519 auth pattern — implement it as a helper class/function within each file (do NOT create a shared module, per user decision).

**Ed25519 Auth Helper (inline in each file):**
- Use Python `cryptography` library for Ed25519 signing (it handles DER/PKCS8/SPKI natively)
- Load private key from `~/.agentbnb/private.key` (DER PKCS8 format, same as Node.js `signing.ts`)
- Load public key from `~/.agentbnb/public.key` (DER SPKI format)
- Sign payload: canonical JSON (sorted keys) of `{ method, path, publicKey, timestamp }`
- `publicKey` value in payload = hex-encoded DER SPKI bytes
- Signature format: base64url encoding
- Return 3 headers: `X-Agent-PublicKey` (hex), `X-Agent-Signature` (base64url), `X-Agent-Timestamp` (ISO 8601)
- Config dir override via `AGENTBNB_DIR` env var (same as Node.js CLI)

**Dependencies: `httpx` and `cryptography` only.** Add a comment at top of each file listing: `# Requirements: pip install httpx cryptography`

**LangChain adapter (`adapters/langchain/agentbnb_tools.py`):**
- Import from `langchain_core.tools import BaseTool` (NOT `langchain.tools` — that is deprecated)
- Import `pydantic.Field` for input schemas
- Class `AgentBnBDiscover(BaseTool)`:
  - name = "agentbnb_discover"
  - description = "Search for AI agent capabilities on the AgentBnB network"
  - Input: `query: str` (Pydantic field)
  - Calls `GET /cards?q={query}&limit=10`
  - Returns JSON list of matching cards (name, id, description, pricing)
- Class `AgentBnBRequest(BaseTool)`:
  - name = "agentbnb_request"
  - description = "Request an AI agent capability execution on AgentBnB"
  - Input: `card_id: str`, `credits: int`, `params: dict` (optional)
  - Calls `POST /api/credits/hold` then the card's gateway (or returns hold confirmation for now since direct execution requires relay). For v1, just do the credit hold and return the escrow receipt — full execution requires relay which is out of scope for a thin wrapper.
  - Actually, simplify: call `POST /api/credits/hold { amount, cardId }` and return the escrow ID + hold confirmation. The README should explain that execution happens via relay/gateway separately.
  - Wait — even simpler and more useful: make `request` do a search + hold in one call. Input: `query: str`, `budget: int`. Search for cards, pick cheapest match, hold credits, return card info + escrow ID.
- Class `AgentBnBStatus(BaseTool)`:
  - name = "agentbnb_status"
  - description = "Check your AgentBnB credit balance and identity"
  - No input required
  - Calls `GET /api/credits/{owner}` (derive owner from public key hex)
  - Returns balance info
- All tools use `_run()` (sync, using `httpx.Client`) and `_arun()` (async, using `httpx.AsyncClient`)
- Constructor takes `registry_url: str` (default `http://localhost:3000`), `config_dir: str | None` (default `~/.agentbnb`)

**CrewAI adapter (`adapters/crewai/agentbnb_tool.py`):**
- Use `from crewai.tools import tool` decorator pattern
- Three module-level functions decorated with `@tool`:
  - `agentbnb_discover(query: str) -> str` — search cards
  - `agentbnb_request(query: str, budget: int) -> str` — search + hold credits
  - `agentbnb_status() -> str` — check balance
- Module-level config: `REGISTRY_URL` from env var `AGENTBNB_REGISTRY_URL` (default `http://localhost:3000`), `CONFIG_DIR` from `AGENTBNB_DIR` (default `~/.agentbnb`)
- Use sync `httpx.Client` (CrewAI tools are sync)
- Return JSON strings (CrewAI tools return str)

Create the OpenAI function calling adapter as a static JSON file plus a TypeScript generator script.

**Generator script (`adapters/openai/generate.ts`):**
- Standalone TypeScript script runnable with `npx tsx adapters/openai/generate.ts`
- Fetches `GET /api/openapi/gpt-actions?server_url=http://localhost:3000` from a running registry
- Transforms each OpenAPI operation into an OpenAI function calling schema entry:
  ```json
  {
    "name": "<operationId>",
    "description": "<operation summary + description>",
    "parameters": {
      "type": "object",
      "properties": { ... from OpenAPI requestBody or query params },
      "required": [ ... ]
    }
  }
  ```
- Writes output to `adapters/openai/functions.json`
- Accept `--registry-url` CLI arg (default `http://localhost:3000`)
- Use plain `fetch()` (no dependencies needed, Node 20+ has global fetch)
