## Decisions Made

- Used `vi.stubGlobal('fetch', ...)` for unit tests — avoids new test dependencies, matches Vitest idioms
- Interleave mode (hasQuery=true) alternates zip from local and deduplicated-remote sublists; local wins dedup in both modes
- `config set` validates key against `allowlist = ['registry']` to prevent arbitrary key pollution
- Explicit `--registry` flag failure exits 1 (user expectation); `config.registry` default failure warns and falls back to local results only
- Source column only added to table when remote results are present — local-only discover output format preserved exactly (no regressions in existing 27 CLI tests)
- `--tag` applied client-side for local results (the GET /cards endpoint has no tag query param)
