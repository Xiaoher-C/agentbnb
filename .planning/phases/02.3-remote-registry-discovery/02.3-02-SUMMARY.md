## Decisions Made
- Spawn registry server as background subprocess (not in-process Fastify): vitest's worker isolation prevents execSync children from accessing loopback ports opened by the test process itself. Background subprocess bypasses this constraint.
- Port-file polling pattern for server-ready signal: server writes its ephemeral port to a temp file once listening; beforeEach polls until file appears (10s deadline).
- Shell redirect `2>&1` for the graceful-degradation test: execSync returns only stdout on exit 0, discarding stderr. Using a shell-level redirect merges streams so the "Cannot reach" warning is captured.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Server fixture changed from in-process to background subprocess**
- **Found during:** Task 1 (Add integration tests for remote registry discovery)
- **Issue:** In-process Fastify server (run inside the vitest test process) is not reachable from CLI subprocesses launched via execSync, even on 127.0.0.1. macOS sandbox blocks loopback access from execSync children to the parent's listening sockets when vitest runs in worker mode.
- **Fix:** Created `test-registry-server.ts` standalone fixture; tests spawn it as a detached background process and poll for a port-file to detect readiness. The subprocess is terminated in afterEach.
- **Files modified:** src/cli/index.test.ts, src/cli/test-registry-server.ts (new)
- **Verification:** All 8 integration tests pass; direct fetch from parent still works in isolation tests
- **Committed in:** 8803680 (Task 1 commit)

**2. [Rule 1 - Bug] Graceful-degradation test uses shell redirect instead of runCli stderr**
- **Found during:** Task 1 (integration test validation)
- **Issue:** The shared `runCli` helper uses `execSync` which discards stderr when exit code is 0; the graceful-degradation test needs to verify the "Cannot reach" warning that goes to console.error (stderr).
- **Fix:** Single test uses direct `execSync` with `2>&1` shell redirect to capture both streams, verifying the warning appears in combined output.
- **Files modified:** src/cli/index.test.ts
- **Verification:** Test passes, warning is captured
- **Committed in:** 8803680 (Task 1 commit)
