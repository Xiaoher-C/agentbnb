  <action>
    Add a new `describe('CLI: discover --registry (integration)', ...)` block at the end of index.test.ts.

    Test fixture (beforeEach/afterEach):
    - Create tmpDir with mkdtempSync, run `runCli('init --owner test-agent', tmpDir)`
    - Open an in-memory database: `openDatabase(':memory:')`
    - Insert a test card with a known UUID and name (e.g., id: 'remote-test-uuid-1234', name: 'Remote Voice Synth')
    - Create registry server: `createRegistryServer({ registryDb: db, silent: true })`
    - Listen on ephemeral port: `await server.listen({ port: 0, host: '127.0.0.1' })`
    - Extract port: `(server.server.address() as import('net').AddressInfo).port`
    - afterEach: `await server.close()`, `db.close()`, `rmSync(tmpDir, ...)`

    Tests:
    1. **"discovers cards from remote registry via --registry flag"**
       - `runCli('discover --registry http://127.0.0.1:${port}', tmpDir)`
       - Expect status 0
       - Expect stdout contains 'Remote Voice Synth'
       - Expect stdout contains '[remote]'

    2. **"discover --registry --json includes source field"**
       - `runCli('discover --registry http://127.0.0.1:${port} --json', tmpDir)`
       - Parse stdout as JSON array
       - Expect at least one card with `source === 'remote'` and name 'Remote Voice Synth'

    3. **"discover --registry with --tag filters results"**
       - Insert a second card with metadata.tags: ['nlp'] into the test DB
       - `runCli('discover --registry http://127.0.0.1:${port} --tag nlp', tmpDir)`
       - Expect stdout contains the tagged card name
       - (Note: if server doesn't filter by tag, client-side filter should still work)

    4. **"shows error when explicit --registry is unreachable"**
       - `runCli('discover --registry http://127.0.0.1:19999', tmpDir)`
       - Expect status !== 0
       - Expect stderr or stdout contains 'Cannot reach'

    5. **"config set registry + discover uses saved default"**
       - `runCli('config set registry http://127.0.0.1:${port}', tmpDir)` — expect status 0
       - `runCli('discover', tmpDir)` — expect stdout contains 'Remote Voice Synth' and '[remote]'

    6. **"config default registry unreachable degrades to local results"**
       - `runCli('config set registry http://127.0.0.1:19999', tmpDir)` — set bad URL
       - Publish a local card first: write card JSON, `runCli('publish card.json', tmpDir)`
       - `runCli('discover', tmpDir)` — expect status 0 (graceful), expect stdout contains local card name, expect stderr contains warning about cannot reach

    7. **"config get registry returns saved value"**
       - `runCli('config set registry http://example.com:7701', tmpDir)`
       - `runCli('config get registry', tmpDir)` — expect stdout contains 'http://example.com:7701'

    8. **"config set unknown key is rejected"**
       - `runCli('config set foobar value', tmpDir)` — expect status !== 0, expect stderr or stdout contains 'Unknown config key'

    Import additions at top of file:
    - `import { createRegistryServer } from '../registry/server.js'`
    - `import { openDatabase, insertCard } from '../registry/store.js'`
    - `import type { AddressInfo } from 'node:net'`

    For the test card inserted into the in-memory DB, construct a full valid CapabilityCard object (not using makeCardJson which returns a string for file writing):
