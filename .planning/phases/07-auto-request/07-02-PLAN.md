  <action>
    Create `src/autonomy/auto-request.ts` following the research pattern (07-RESEARCH.md Pattern 1-4):

    1. **Interfaces:**
       ```typescript
       export interface CapabilityNeed {
         query: string;
         maxCostCredits: number;
         params?: Record<string, unknown>;
       }

       export interface AutoRequestOptions {
         owner: string;
         registryDb: Database.Database;
         creditDb: Database.Database;
         autonomyConfig: AutonomyConfig;
         budgetManager: BudgetManager;
         maxSearchResults?: number;  // default 10
       }

       export interface AutoRequestResult {
         status: 'success' | 'budget_blocked' | 'tier_blocked' | 'no_peer' | 'failed';
         result?: unknown;
         escrowId?: string;
         peer?: string;
         creditsSpent?: number;
         reason?: string;
       }
       ```

    2. **Helper functions (exported for testing):**
       - `minMaxNormalize(values: number[]): number[]` — guard max===min returns all 1s
       - `scorePeers(candidates, selfOwner): ScoredPeer[]` — extract success_rate (default 0.5), cost efficiency (0 cost -> inverse 1), idle_rate (missing _internal -> 1.0); normalize each dimension; multiply; sort descending

    3. **AutoRequestor class:**
       ```typescript
       export class AutoRequestor {
         constructor(opts: AutoRequestOptions) { ... }
         async requestWithAutonomy(need: CapabilityNeed): Promise<AutoRequestResult> { ... }
       }
       ```

       The `requestWithAutonomy` flow:
       a. `searchCards(registryDb, need.query, { online: true })` to find candidates
       b. Filter: `card.owner !== this.owner` (self-exclusion, REQ-03)
       c. For v2.0 cards, flatten skills[] to get individual skill candidates; filter by `skill.pricing.credits_per_call <= need.maxCostCredits`
       d. For v1.0 cards (no skills[]), use the card-level pricing; filter by cost cap
       e. If no candidates after filtering: log failure via `insertAuditEvent` with `auto_request_failed`, return `{ status: 'no_peer' }`
       f. Score candidates via `scorePeers()`, pick top scorer
       g. Find peer gateway config: `findPeer(topScorer.card.owner)`. If no peer config found: log failure, return `{ status: 'no_peer', reason: 'No gateway config for peer' }`
       h. `getAutonomyTier(cost, this.autonomyConfig)`:
          - Tier 3: `createPendingRequest(registryDb, ...)`, log audit event, return `{ status: 'tier_blocked' }`
          - Tier 1 or 2: continue to execution
       i. `budgetManager.canSpend(cost)`: if false, log failure, return `{ status: 'budget_blocked' }`
       j. `holdEscrow(creditDb, this.owner, cost, card.id)`
       k. `requestCapability({ gatewayUrl, token, cardId, params: { skill_id, ...need.params } })`
          - On success: `settleEscrow(creditDb, escrowId, card.owner)`, return `{ status: 'success', ... }`
          - On error: `releaseEscrow(creditDb, escrowId)`, log failure, return `{ status: 'failed', ... }`
       l. If Tier 2: also call `insertAuditEvent` with `auto_request_notify` after successful execution

    4. **Failure logging (REQ-06):** Every non-success path calls `insertAuditEvent(registryDb, { type: 'auto_request_failed', ... })`. For no_peer/budget_blocked where no specific card is selected, use card_id='system', skill_id='none', peer='none'.

    5. **Tests in `src/autonomy/auto-request.test.ts`:**

         - Test all result statuses: no_peer, budget_blocked, tier_blocked, success, failed
         - Verify request_log has audit entries for failures (query request_log table directly)
         - Verify pending_requests table has row for tier_blocked case

         // Load config
