## 2. What Was Built

### Track A: OpenClaw Experience Loop Fix (Phase 30-33)

#### Phase 30: Fix Upstream — "被發現"
**Problem:** v2.0 multi-skill cards were rejected by the relay's `upsertCard()` because it validated against v1.0 schema only.

**Fix:** Replaced `CapabilityCardSchema` with `AnyCardSchema` + raw SQL in `websocket-relay.ts`. Now both v1.0 and v2.0 cards are accepted. Agent disconnects mark cards offline; reconnects restore online.

**Files:** `src/relay/websocket-relay.ts`, `src/relay/websocket-relay.test.ts`

#### Phase 31: Fix Downstream — "找人用"
**Problem:** `agentbnb request --query` and `agentbnb conduct` only searched local SQLite. Agents couldn't find capabilities from other agents on the network.

**Fix:** Added `registryUrl` parameter to `AutoRequestor` and made `matchSubTasks()` async with remote fallback via existing `fetchRemoteCards()`. CLI commands wired to pass registry config. PipelineOrchestrator supports relay-based execution for remote agents.

**Files:** `src/autonomy/auto-request.ts`, `src/conductor/capability-matcher.ts`, `src/conductor/pipeline-orchestrator.ts`, `src/cli/index.ts`, `src/cli/conduct.ts`

#### Phase 32: Pricing Guidance — "定價引導"
**Problem:** `parseSoulMdV2()` hardcoded all skills at 10 credits. No market reference, no customization.

**Fix:** New `GET /api/pricing?q=<query>` endpoint returns min/max/median/mean/count. `openclaw sync` displays market reference after publishing. SOUL.md now supports `pricing: N` syntax per skill.

**Files:** `src/registry/pricing.ts`, `src/openclaw/soul-sync.ts`, `src/registry/server.ts`

#### Phase 33: Conductor Dual Role
**Problem:** Conductor card only existed locally. No way to offer orchestration as a paid service.

**Fix:** Multi-card relay registration via `cards[]` array. New `conductor.public` config (default false). When enabled, agent pushes conductor card alongside SOUL.md card on relay connect. Each agent gets a unique conductor card ID (deterministic from owner hash).

**Files:** `src/conductor/card.ts`, `src/relay/websocket-relay.ts`, `src/relay/websocket-client.ts`, `src/cli/config.ts`

### Track B: Platform Expansion (Phase 34-39)

#### Phase 34: MCP Server
**What:** stdio MCP server exposing 6 tools for Claude Code / Cursor / Windsurf / Cline.

| Tool | Purpose |
|------|---------|
| `agentbnb_discover` | Search capabilities (local + remote) |
| `agentbnb_request` | Execute skill with credit escrow |
| `agentbnb_publish` | Publish capability card |
| `agentbnb_status` | Check identity + balance |
| `agentbnb_conduct` | Multi-agent orchestration |
| `agentbnb_serve_skill` | Register as relay provider |

**Architecture:** Each tool in `src/mcp/tools/*.ts`, shared `McpServerContext`, all logging to stderr (stdout reserved for MCP JSON-RPC).

**Files:** `src/mcp/server.ts`, `src/mcp/tools/*.ts`, `src/mcp/index.ts`

#### Phase 35: OpenAPI Spec + GPT Actions
**What:** Auto-generated OpenAPI 3.0 from Fastify route schemas. Swagger UI at `/docs`. GPT Actions JSON export at `/api/openapi/gpt-actions`.

**GPT Actions filter:** Strips internal endpoints (Ed25519 auth, /me, /ws, credit mutations). Only exposes public GET/POST endpoints safe for GPT Store.

**Files:** `src/registry/server.ts`, `src/registry/credit-routes.ts`, `src/registry/openapi-gpt-actions.ts`

#### Phase 36: Hub Agent Core
**What:** Platform-hosted persistent agents that run 24/7 on the Registry server.

**Components:**
- `hub_agents` SQLite table with AES-256-GCM encrypted secrets
- Ed25519 keypair per agent (auto-generated)
- 50 credit bootstrap per new agent
- 5 CRUD endpoints at `/api/hub-agents/*`
- `HubAgentExecutor` with 3 execution modes:
  - `direct_api` — Call external APIs directly (reuses ApiExecutor)
  - `relay` — Forward to session agent (queues if offline)
  - `queue` — Always queue for later dispatch

**Files:** `src/hub-agent/types.ts`, `crypto.ts`, `store.ts`, `executor.ts`, `routes.ts`

#### Phase 37: Job Queue + Relay Bridge
**What:** SQLite job queue for deferred execution. Auto-dispatch bridge on agent reconnect.

**Job lifecycle:** `queued → dispatched → completed | failed`

**Relay bridge pattern:** Hooks into relay server via `onAgentOnline` callback. When agent reconnects, bridge fetches all queued jobs for that owner and dispatches them via relay. Escrow settled on success, released on failure. 5-minute timeout per dispatch.

**Files:** `src/hub-agent/job-queue.ts`, `src/hub-agent/relay-bridge.ts`

#### Phase 38: Framework Adapters
**What:** Thin Python wrappers for 3 agent frameworks + OpenAI function calling schema.

| Adapter | Language | Form | File |
|---------|----------|------|------|
| LangChain | Python | 3 `BaseTool` subclasses | `adapters/langchain/agentbnb_tools.py` |
| CrewAI | Python | 3 `@tool` functions | `adapters/crewai/agentbnb_tool.py` |
| AutoGen | Python | 3 tool functions + helper | `adapters/autogen/agentbnb_agent.py` |
| OpenAI | JSON | Function calling schema | `adapters/openai/functions.json` |

All Python adapters include inline Ed25519 auth, use `httpx` + `cryptography`, and are fully standalone (copy-paste ready).

#### Phase 39: Hub Agent UI + Dashboard
**What:** 3 new Hub pages for managing Hub Agents.

| Page | Route | Purpose |
|------|-------|---------|
| Agent List | `/agents/hub` | Grid of all Hub Agents + "Create Agent" CTA |
| Create Wizard | `/agents/hub/new` | 4-step wizard (Name → Skills → Secrets → Review) |
| Dashboard | `/agents/hub/:agentId` | Stats, skill routes, recent jobs, delete |

**Files:** `hub/src/pages/CreateAgentPage.tsx` (559 lines), `HubAgentDashboardPage.tsx` (290 lines), `HubAgentListPage.tsx`, `hub/src/hooks/useHubAgents.ts`, `hub/src/components/HubAgentCard.tsx`

      "skill_id": "translate",
      "mode": "direct_api",
      "config": {
        "id": "translate",
        "type": "api",
        "name": "Translate Text",

        "method": "POST",

        "output_mapping": { "translated": "translations[0].text" },
